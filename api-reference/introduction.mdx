The endpoint reference is generated from `api-reference/openapi.yaml`, which describes the live Faye Partner Sales API.

Use the guide pages for implementation sequence and business rules. Use the generated reference for exact request and response schemas.

## Environments

| Environment | Base URL |
| --- | --- |
| Production | `https://partnerapi.withfaye.com` |
| Staging | `https://partnerapi.fayehq.com` |
| Development | `https://partnerapi.faye-dev.com` |

## Authentication

Most endpoints require bearer authentication.

```http
Authorization: Bearer <token>
```

In the interactive API playground, paste only the token value into the authorization field. The OpenAPI security scheme adds the bearer prefix.

## Endpoint groups

| Group | Use it for |
| --- | --- |
| Quote | Create, recalculate, and retrieve single or multi-product quotes. Start with [`POST /v2/quote`](/api-reference/quote/generate-a-new-quote-v2-single-product). |
| Purchase | Save and pay for a quote, then issue the policy. Start with [`POST /v2/purchase`](/api-reference/purchase/save-and-pay-for-a-quote-v2). |
| Claims | Submit claims, upload claim files, track status, and receive webhook updates. Start with [Claims API](/api-reference/claims-overview). |
| Advisor | Create, update, or delete advisor profiles under your partner account. Start with [`POST /v1/advisor`](/api-reference/advisor/create-or-update-an-advisor). |
| Policy | Retrieve issued policy details with [`GET /v1/policy/code/{policyCode}/email/{email}`](/api-reference/policy/get-policy-by-policycode-and-email). |
| Product | Retrieve product definitions with [`GET /v1/product`](/api-reference/product/get-all-products) when your partner configuration supports it. |
| Amendment | Quote amendments, apply policy changes, or cancel a policy. Start with [`POST /v1/amendment`](/api-reference/amendment/calculate-the-amendment-cost). |
