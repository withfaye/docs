This guide shows the recommended V2 flow for new partner integrations.

<Info>
  Replace the sample dates, traveler details, product IDs, and tokens with values from your Faye onboarding package.
</Info>

## 1. Send authenticated requests

Faye gives each partner an environment-specific bearer token. Send it in the `Authorization` header.

Open the generated endpoint page for [`POST /v2/quote`](/api-reference/quote/generate-a-new-quote-v2-single-product) when you want to test this request in the API playground.

```bash
export FAYE_BASE_URL="https://partnerapi.fayehq.com"
export FAYE_API_TOKEN="YOUR_STAGING_TOKEN"
```

```bash
curl "$FAYE_BASE_URL/v2/quote" \
  -H "Authorization: Bearer $FAYE_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "tripStartDate": "2026-08-15",
    "tripEndDate": "2026-08-22",
    "destinations": [
      {
        "country": "FR",
        "label": "France"
      }
    ],
    "residency": {
      "country": "US",
      "areaLevel1": "CA"
    },
    "travellers": [
      {
        "firstName": "Jane",
        "lastName": "Doe",
        "email": "jane.doe@example.com",
        "dateOfBirth": "1985-05-15",
        "mainTraveller": true,
        "isDriver": false
      }
    ],
    "product": "PR-STANDARD-01",
    "tripCost": 5200,
    "coverageOptions": {
      "trcOption": {
        "selected": true
      }
    },
    "timezone": "America/Los_Angeles",
    "externalId": "booking_12345"
  }'
```

## 2. Store the quote hash and price

The quote response includes the price, add-on availability, policy document links, and `quoteHash`.

```json
{
  "currentPrice": 375.35,
  "quoteHash": "abc123def456",
  "policyExampleUrl": "https://example.com/sample-policy.pdf",
  "quoteUrl": "https://example.com/quote.pdf",
  "quotePageUrl": "https://example.com/quote"
}
```

Before purchase, show the traveler the price and make the `policyExampleUrl` available for review.

## 3. Tokenize the credit card

Collect payment details with your Faye-approved payment provider. For Checkout.com, the provider returns a single-use token such as `tok_...`. For Stripe, it returns a token, payment method, or payment intent depending on your Faye configuration.

Do not send raw card numbers or CVV values to Faye.

```js
const tokenResponse = await tokenizeCardWithYourProvider();

const paymentToken = tokenResponse.token;
```

See [Card tokenization](/api-reference/card-tokenization) for provider-specific examples.

## 4. Purchase the quote

Send the quote hash and payment token to [`POST /v2/purchase`](/api-reference/purchase/save-and-pay-for-a-quote-v2).

```bash
curl "$FAYE_BASE_URL/v2/purchase" \
  -H "Authorization: Bearer $FAYE_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "policyCodeHash": "abc123def456",
    "paymentInfo": {
      "paymentService": "checkOut",
      "paymentToken": "tok_checkout_single_use_token"
    },
    "osInfoDto": {
      "customerTZ": "America/Los_Angeles",
      "operationSystem": "web",
      "osVersion": "Chrome"
    }
  }'
```

The response confirms the issued policy.

```json
{
  "purchaseId": "P26-1234",
  "plans": [
    {
      "name": "Jane Doe",
      "planId": "P26-1234-1"
    }
  ],
  "totalPrice": 375.35,
  "policyUrl": "https://example.com/policy.pdf",
  "receiptUrl": "https://example.com/receipt.pdf"
}
```

## 5. Handle common failures

| Status | Meaning | What to do |
| --- | --- | --- |
| `400` | The quote or purchase request failed validation. | Show the returned message and let the traveler correct the trip, traveler, residency, add-on, or payment inputs. |
| `401` | The bearer token is missing or invalid. | Check the environment and token. Staging tokens do not work in production. |
| `402` | Payment failed. | Let the traveler retry with another card or payment method. |

For all purchase calls, treat a `Price Mismatch` response as a hard stop. Recreate or recalculate the quote, show the updated price, then retry payment.

After purchase, retrieve the policy with [`GET /v1/policy/code/{policyCode}/email/{email}`](/api-reference/policy/get-policy-by-policycode-and-email).
